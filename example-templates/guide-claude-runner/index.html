Arquitectura

transcript  ──►  agente  ──►  datos  ──►  editor  ──►  reporte
              │
              ├── claude.ai · projects     (recomendado)
              ├── claude code (CLI)
              └── API de Anthropic         (próxima etapa)

Las tres opciones del agente devuelven la misma estructura de datos, así que se puede empezar con una y migrar a otra sin rehacer briefs anteriores.

Claude.ai · Projects Recomendado

Cómo configurar · una sola vez

  1. Crear un Project en claude.ai
  2. Pegar el contenido de system-prompt.md en las instrucciones del Project
  3. Subir cuestionario-onboarding.md y output-schema.md como knowledge

Cómo usar · cada reunión

  1. Abrir una conversación nueva dentro del Project
  2. Pegar el transcript y enviar
  3. Copiar la respuesta y abrirla en el editor (sección 03)
Claude Code (CLI)

Útil cuando alguien del equipo ya usa Claude Code para otras tareas. Corre el agente y, opcionalmente, el renderer en la misma sesión local.

API de Anthropic No implementado

Integración directa con la API, orquestada con Make o un sistema propio. El transcript llega por una integración y el brief sale sin intervención manual.

05

Notas

  • El PDF es solo lectura: no contiene los datos y no se puede reabrir en el editor. Para preservar la posibilidad de editar más adelante, hay que guardar el HTML exportado (o la respuesta original del agente). El PDF sirve para enviar; el HTML, para archivar.
  • Si cambia el contenido del brief (qué bloques aparecen, qué campos pide), hay que actualizar tanto el agente como el editor. No es algo que cambie seguido.
  • El editor es público: cualquiera con el link puede abrirlo. Si en algún momento se necesita restringir el acceso, se resuelve a nivel del hosting.
